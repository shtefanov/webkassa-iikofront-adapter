# Webkassa Support Request: XTIN Field

Date: 02-07-2026

Subject: WebNKT / API 2.0.3: how should integrators pass XTIN in `/api/v4/check`?

Здравствуйте.

Разрабатываем интеграцию с Webkassa API v4, протокол `2.0.3`, для фискализации
чеков из iikoFront.

Из официальной Postman-документации по `/api/v4/check` в позиции чека мы нашли
поля:

- `GTIN` - глобальный идентификатор товара / штрихкод;
- `NTIN` - национальный идентификатор товара / штрихкод;
- `ProductId` - ID продукта на виртуальном складе;
- `WarehouseType` - признак виртуального склада.

Также в публичных материалах Webkassa по WebNKT указано, что WebNKT может
получать постоянный `NTIN` или временный `XTIN`.

Просим уточнить:

1. Должен ли интегратор когда-либо передавать временный `XTIN` в запросе
   `/api/v4/check`?
2. Если да, в каком JSON-поле позиции нужно передавать `XTIN`?
3. Если отдельного поля `XTIN` нет, нужно ли временный XTIN передавать в поле
   `NTIN`, или интегратор должен передавать только `GTIN`/`NTIN`, а временный
   `XTIN` формируется и передаётся WebNKT внутренне?
4. Для товаров без виртуального склада правильно ли передавать только `GTIN`
   или `NTIN` без `WarehouseType=1`?
5. Есть ли отдельные требования WebNKT к возвратам продажи: нужно ли в возврате
   повторно передавать `GTIN`/`NTIN` по позициям?

Нам нужно корректно настроить field map интеграции:

```json
{
  "GTIN": "...",
  "NTIN": "...",
  "ProductId": "...",
  "WarehouseType": 0
}
```

Спасибо.
